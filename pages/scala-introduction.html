Introduction

SBT and editor setup

  • Install sbt following either
    • installing sbt on windows
    • or installing sbt on linux including a linux subsystem on Windows 10
  • Get a zip file
    • Unzip it and change directory to root-dir-for-the-course .
    • Make sure that typing sbt run from the sbt-test directory correctly runs the test script. 
       cd sbt-test
 sbt run
  • Explore some of the directories in this directory. In particular, find the code fragments automatically extracted from each chapter, and the complete runnable examples. Examine the complete runnable HelloWorld example from Chapter 1.

(Note that no build.sbt file or fancy directory structure is required for a simple single-file project with no dependencies.)

Create your own Scala SBT project.

  • Change directory to scala-course-2018/root-dir-for-the-course
  • Try not to run SBT from the app-template directory in order to keep it clean.
  • Copy that directory (and its contents) as exercises/C1-HelloWorld and create your own SBT project.
  • Just copy the source code file (examples/C1-HelloWorld/HelloWorld.scala) for the HelloWorld example into the correct source code sub-directory (src/scala/C1-HelloWorld/HelloWorld.scala) and check that it works by first running sbt and then run from the SBT prompt.
  • Open up the source code in your editor/IDE and edit the message that is printed, save, then run again from SBT.
  • Then type ~run in SBT, go back to your editor and change the message again. As you save the buffer, note that SBT detects that the source file has changed and automatically re-compiles and re-runs the project.
  • Use the REPL
    • From the SBT prompt, type console to get a REPL. Enter 1+2 to check it works.
    • To exit REPL, type 'ctrl-D'.
